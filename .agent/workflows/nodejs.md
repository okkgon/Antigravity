# Node.js 모듈 생성 워크플로우

이 워크플로우는 프로젝트의 최신 JavaScript 개발 규칙을 준수하여 표준 Node.js 모듈과 그에 해당하는 테스트 파일을 생성합니다.

## 단계

1. **모듈 세부 정보 요청:**
   사용자에게 모듈의 의도된 기능과 이름(예: `userController`, `mathUtils`)을 묻고, CommonJS(`require`) 방식과 ES Modules(`import/export`) 방식 중 어느 것을 선호하는지 확인합니다. 사용자의 응답을 기다립니다.

2. **핵심 로직 파일 생성:**
   지정된 디렉토리에 모듈의 기본 JavaScript 파일(예: `[moduleName].js`)을 생성합니다.
   - 단일 책임 원칙(Single Responsibility)으로 로직을 설계합니다.
   - 비동기 작업에 주로 `async`/`await`를 활용하여 함수형 접근 방식을 정의합니다.
   - 견고한 에러 처리를 위해 `try...catch` 블록을 적용합니다.
   - `const`/`let` 및 화살표 함수, 구조 분해 할당 등 최신 문법을 사용합니다.

3. **단위 테스트 파일 생성:**
   해당 모듈을 위한 테스트 파일(예: `[moduleName].test.js`)을 생성합니다.
   - 모듈 함수를 가져와서 테스트하는 기본 테스트 케이스를 구현합니다.
   - 프레임워크가 지정되지 않은 경우 Node 내장 `assert` 모듈을 사용하거나, Jest와 같은 테스트 프레임워크용 표준 구조를 설정합니다.

4. **작업 공간에 파일 저장:**
   `write_to_file` 도구를 사용하여 대상 Node.js 프로젝트 또는 튜토리얼 폴더 내에 핵심 로직 파일과 테스트 파일을 저장합니다.

5. **사용자 알림:**
   사용자에게 모듈 및 테스트 파일 준비가 완료되었음을 알리고, 새 테스트를 실행하거나 모듈을 통합하는 방법을 설명합니다.
